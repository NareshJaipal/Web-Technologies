Web Technologies

The focus in this course is on the World Wide Web as a platform for interactive applications, content publishing and social services.

  1. HTML (Hypertext Markup Language)
    • HTML structures web content using elements like headings, paragraphs, images, and links.
    • It provides semantic meaning to content, defining the layout and hierarchy of information.
  2. CSS (Cascading Style Sheets)
    • CSS is a style sheet language used for describing the presentation and visual layout of HTML elements on a web page.
    • It allows developers to control the appearance of elements by specifying attributes like colors, fonts, spacing, layout, and more.
  3. JavaScript
    • Enhances interactivity on web pages by allowing dynamic content updates, event handling, and DOM manipulation.
    • Enables the creation of engaging user experiences by adding functionality such as form validation, animations, and AJAX requests.

Routing Concepts

TRouting concepts in the context of computer networks or web applications involve directing data packets or controlling the flow of application requests between different nodes or endpoints.

  1. Routing Algorithms
    • These are the methodologies or protocols used to determine the best path for data packets to travel from a source to a destination in a network. Examples include OSPF (Open Shortest Path First), RIP (Routing Information Protocol), BGP (Border Gateway Protocol), and more.
    • Algorithms consider factors like network topology, link cost, traffic load, and latency to determine the optimal path for data transmission.
  2. IP Routing
    • In the context of the Internet Protocol (IP), routing involves the process of forwarding IP packets from their source to the intended destination across interconnected networks.
    • IP routing utilizes routing tables, which contain information about available paths and network addresses, enabling routers to make decisions on how to forward packets efficiently.
  3. Routing Protocols and Policies
    • Routing protocols define the rules and standards used by routers to communicate and exchange routing information.
    • Routing policies determine how data traffic is directed within a network, including rules for prioritization, access control, load balancing, and network segmentation.

System & Network Administration

System and network administration involve managing and maintaining the infrastructure, systems, and networks that enable the operation and connectivity of computers, servers, and devices within an organization.

  1. System Administration
    • User Management: Creating and managing user accounts, permissions, and access levels to various systems and resources.
    • System Configuration: Installing, configuring, and maintaining operating systems (such as Linux, Windows, Unix) on servers and computers.
    • Software Installation and Updates: Managing software installations, updates, patches, and ensuring compatibility and security.
    • Backup and Recovery: Implementing backup strategies and disaster recovery plans to protect data and ensure business continuity.
  2. Network Administration
    • Network Configuration: Setting up and maintaining network devices, such as routers, switches, firewalls, and access points, to ensure connectivity and security.
    • IP Address Management (IPAM): Assigning and managing IP addresses, subnets, and DHCP (Dynamic Host Configuration Protocol) settings to devices on the network.
    • Network Security: Implementing security measures like firewalls, intrusion detection/prevention systems (IDS/IPS), VPNs (Virtual Private Networks), and access controls to safeguard the network against cyber threats.

Modeling & Simulation

Modeling and simulation are computational techniques used across various fields to replicate, mimic, or represent real-world systems or processes.

  1. Modeling
    • Types of Models: There are various types of models used in different domains, such as mathematical models, physical models, conceptual models, and computer-based models.
    • Purpose of Modeling: Models are used to represent, analyze, predict, and understand real-world systems or phenomena. They help in simplifying complex systems to study their behavior and make predictions.
  2. Simulation
    • Simulation Process: Simulation involves running a model over time to observe and analyze its behavior or outcomes. It allows the experimentation and analysis of a system's behavior without impacting the real system.
    • Simulation Tools and Software: Various software tools and programming languages (such as MATLAB, Simulink, AnyLogic, Arena, Python with libraries like SimPy) are used to create and run simulations.

Software Engineering

Software engineering involves the application of disciplined and systematic approaches to the design, development, testing, deployment, and maintenance of software systems.

  1. Software Development Life Cycle (SDLC)
    • Phases: SDLC comprises phases like requirement analysis, design, implementation, testing, deployment, and maintenance.
    • Models: Various models like Waterfall, Agile, Scrum, and DevOps guide the software development process, each with its approach to managing projects.
  2. Requirements Engineering
    • Gathering Requirements: Understanding and documenting user needs and system requirements before software development begins.
    • Analysis and Specification: Analyzing, prioritizing, and defining requirements in a detailed and clear manner.
